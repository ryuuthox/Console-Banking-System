# 🏦 Console-Based Banking System 

A robust, real-world banking application built entirely in C++. This project serves as a capstone for my Object-Oriented Programming (OOP) and System Architecture studies, focusing heavily on the "Engine" (Back-End logic) rather than the interface.

## 🚀 Project Overview
This system simulates a real banking environment. All user interactions occur through a clean Command Line Interface (CLI), allowing the underlying C++ logic, memory management, and data structures to shine.

## ⚙️ Core Features
* **Secure Access:** Login system for clients and administrators.
* **Transaction Management:** Handled logic for Deposits, Withdrawals, and total balance updates.
* **Client Management:** Ability to add, delete, update, and search for client records.
* **Data Persistence:** Client data and transaction histories are saved and retrieved efficiently.

## 🧠 OOP & Architecture Applied
This project was built to demonstrate "design readiness" and avoid common overengineering. 
* **Encapsulation:** Protected client balances and sensitive data, accessible only through secure methods.
* **Inheritance & Polymorphism:** Structured user roles (e.g., Admin vs. Client) to share core logic while maintaining strict access permissions.
* **Scalability:** Designed the class structure so new features (like VIP accounts or loan systems) can be added iteratively without breaking existing code.
